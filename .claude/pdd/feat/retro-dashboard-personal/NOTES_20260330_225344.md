## 技術的判断

### SSE 断念 → ポーリング
- `dispatch(msg)` は `Cmd` を返すだけで、`Scheduler::add` で実行されなければ `inbox.push` が発火しない
- JS FFI コールバック内で `dispatch(msg) |> ignore` してもメッセージは届かない
- Rabbita に `Sub` (Subscription) 機構がないため、外部イベント → TEA ループの公式パターンが存在しない
- 10秒ポーリングで回避。常時表示用途には十分

### AI リアクション生成
- `claude -p` をサブプロセスで呼び出し。Max サブスクで使える唯一の方法
- Agent SDK はサブスクリプション認証非対応（API キー必須）
- 初回生成は数十秒かかるが、mtime キャッシュで2回目以降は即座
- JSON 出力をプロンプトで指定し、コードブロック除去で安定パース
