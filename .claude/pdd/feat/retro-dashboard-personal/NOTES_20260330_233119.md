## MoonBit で判明した重要な制約

### 1. `dispatch(msg)` は副作用を起こさない
- `Dispatch[Msg] = (Msg) -> Cmd` は Cmd を返すだけ
- 実際の `inbox.push(msg)` は `Scheduler::add(cmd)` で実行されて初めて発火
- JS FFI コールバック内で `dispatch(msg) |> ignore` してもメッセージは届かない
- **影響**: SSE、WebSocket、任意の非同期イベントから TEA ループにメッセージを送れない
- **回避策**: `@http.get` ポーリング。Rabbita に `Sub` 機構が追加されるまでの暫定

### 2. `Scheduler` trait の internal visibility
- `@cmd.raw_effect` のコールバックで `scheduler.add(cmd)` を呼びたいが、`Scheduler` は `internal/runtime` パッケージ
- `@cmd` が `pub using` で re-export しているが、メソッド呼び出しには元パッケージのインポートが必要
- 外部パッケージからは `internal/` をインポートできない
- **影響**: `@http` モジュールと同じパターンが外部で再現できない

### 3. 構文の注意点
- `using @pkg { items }` — ドットなし
- ラベル引数: 定義 `param~`、呼び出し `param=value`
- `f!(args)` は非推奨、`f(args)` に統一
- HTML 要素は子要素（positional arg）必須、空なら `""`
- `target="_blank"` ではなく `target=Blank`（enum）
- `String.split` は `String` を取る（`Char` ではない）
- 文字列補間内に `"` は使えない（変数に代入してから埋め込む）

### 4. JS FFI
- struct を JS → MoonBit に渡すのは内部表現の不一致で失敗
- プリミティブ型（Double, String, Bool）のみ安全
- 複雑なデータは proxy サーバー + `@http.get` + `FromJson` derive が安定パターン

## Phase 2 で必要なこと

### サーバーリファクタ (Python → Hono/Bun)
- `server.py` が 500行超の単一ファイルに肥大化
- Hono を選ぶ理由: SSE 組み込み (`c.streamSSE`)、TypeScript、Bun で高速起動
- ルート分割: `routes/daily.ts`, `routes/claude.ts`, `routes/metrics.ts`, `routes/feed.ts`
- サービス分割: `services/vault.ts`, `services/prometheus.ts`, `services/reaction.ts`
- `claude -p` の呼び出しは `Bun.spawn` で

### UI リデザイン (VT100 → 初代 Macintosh)
- 現在の緑 CRT は VT100/Apple II の美学で、初代 Mac とは異なる
- 初代 Mac は白背景 + 黒テキスト + GUI + Chicago フォント + 1-bit グラフィクス
- Wokyis M5 の筐体が初代 Mac を模しているため、UI も合わせるのが一貫性がある
- CSS の全面書き換え + アイコンの再設計が必要
- MoonBit 側のコードは view 関数の class 名変更程度で済むはず

## AI リアクション生成の詳細

- `claude -p` をサブプロセスで呼び出し（Max サブスクリプションで使える）
- Agent SDK はサブスクリプション認証非対応（API キー必須、GitHub Issue #559）
- プロンプト: タイムラインを渡して JSON 配列で `{time, reaction}` を返させる
- コードブロック除去（```json ... ``` のラップを strip）
- mtime ベースキャッシュ: 日報ファイルが変更されるまで再生成しない
- 初回は数十秒、2回目以降は即座
